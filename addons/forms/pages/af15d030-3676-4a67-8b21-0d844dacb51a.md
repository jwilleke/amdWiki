# Using FormPlugin

The **FormPlugin** embeds a data-entry form on any page using a form definition stored on the server.

## Embedding a Form

Place the plugin anywhere in a page to render the form:

```
[{Form id='your-form-id'}]
```

The `id` must match the name of a form definition file created by an administrator. If the form is not found, a warning box appears instead.

Two forms can appear on the same page with different `id` values — each submits independently.

## How Forms Work

When a logged-in user opens a page containing a form:

1. Fields marked with a **prefill** source are pre-populated from the user's profile (name, email, phone, unit address). The values are editable — prefill sets a starting default, not a locked value.
2. The user fills in or adjusts the fields and clicks **Submit**.
3. The form submits without a page reload. A success or error message appears inline.
4. On success, the form resets and a confirmation email is sent to the submitter (if email is configured).

Anonymous (not logged-in) users see all fields empty regardless of prefill settings.

## Field Grouping

Administrators can group related fields under labelled headings using `section` field entries. These appear as bordered fieldsets in the rendered form — they are visual only and contribute no data to the submission.

## Submitting on Behalf of Another Person

If the form definition has `"proxySubmission": true`, a **For Another Occupant** section appears at the bottom of the form. This section is optional — leave it blank to submit for yourself.

If you fill in any field in that section, **Full Name** becomes required. The server validates this and returns an inline error if name is missing while other fields are filled.

## Notes

- If the `forms` add-on is not enabled, the plugin returns a server error. Contact your administrator.
- If the form definition file is missing or invalid, a warning box appears — other content on the page is unaffected.
- Time validation: forms with both `startTime` and `endTime` fields reject submissions where end time is not later than start time.
- Calendar conflict detection: if the form is connected to a calendar handler, submitting a slot that is already reserved returns an inline error without a page reload.
- For administrators creating form definitions — field types, prefill paths, proxy submission, and the full JSON schema — see [Form Definition Reference].
