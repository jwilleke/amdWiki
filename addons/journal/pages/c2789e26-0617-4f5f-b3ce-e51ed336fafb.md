# Journal Help

The Journal lets you keep private, dated entries — like a personal diary — directly within your wiki. Entries support full Markdown, attachments, mood tracking, tags, writing templates, and voice-to-text.

## Getting Started

| Action | How |
|--------|-----|
| Create today's entry | Click **New Entry** on the [Journal timeline](/journal), or visit [/journal/new](/journal/new) |
| View your entries | [/journal](/journal) |
| Edit an entry | Open the entry and click **Edit**, or go to `/journal/<slug>/edit` |
| Delete an entry | Open the edit form and click **Delete** |

Entries are **private by default** — only you and administrators can see them.

---

## The Editor

### Templates

When creating a new entry, a template picker appears at the top of the editor. Choose a template to pre-fill the content area with a structured prompt.

| Template | Best for |
|----------|----------|
| **Free Write** | Open-ended journaling — blank canvas |
| **Morning Reflection** | Intentions and gratitude to start the day |
| **Evening Review** | What went well, what to improve, tomorrow's focus |
| **Weekly Review** | Accomplishments, blockers, goals for next week |

Click any card to select it. The content area is only pre-filled if it is currently empty — your own text is never overwritten. The picker collapses after you select a template.

Custom templates can be added by placing `.md` files in `data/journal/templates/`. Each file should start with an HTML comment block:

```
<!-- name: My Template
description: A short description shown in the picker -->

Your template content here...
```

### Mood Picker

Select an optional mood by clicking one of the emoji buttons below the Date field. Click **clear** to remove a mood. Moods appear on your timeline cards and can be used to filter entries.

### Voice to Text

If your browser supports the Web Speech API (Chrome, Edge, Safari), a microphone button appears next to the Content label.

- Click the button to start recording — it pulses red while active.
- Speak naturally; final transcript is appended to the content area.
- Interim (in-progress) text appears next to the button in grey italics.
- Click the button again to stop.

Voice is transcribed locally in the browser — no audio is sent to the server.

### Tags

Enter comma-separated tags in the **Tags** field (e.g. `family, gratitude, work`). Tags appear on timeline cards and can be used to filter entries.

## Browsing Your Journal

### Timeline

[/journal](/journal) shows your entries newest-first with mood, tags, and an excerpt. The sidebar shows:

- **Writing streak** — consecutive days with at least one entry
- **Mood filter** — click any mood to show only matching entries
- **Tag filter** — click any tag to show only matching entries

### Filter by Tag or Mood

| URL | Shows |
|-----|-------|
| `/journal/tag/gratitude` | All your entries tagged `gratitude` |
| `/journal/mood/happy` | All your entries with mood `happy` |

### On This Day

`GET /api/journal/on-this-day` returns entries from the same calendar date in prior years — useful for reflection widgets. The `[{Journal view='on-this-day'}]` plugin (see below) renders this inline on any page.

---

## The `[{Journal}]` Plugin

Embed a live journal widget on any wiki page:

```
[{Journal}]
[{Journal view='timeline' limit='10'}]
[{Journal view='streak'}]
[{Journal view='on-this-day'}]
```

| Parameter | Values | Default | Description |
|-----------|--------|---------|-------------|
| `view` | `timeline` `streak` `on-this-day` | `timeline` | Which widget to render |
| `limit` | number | `10` | Max entries shown (timeline only) |

The widget shows **your own entries only** and respects privacy — it renders nothing for unauthenticated visitors.

## Exporting Your Journal

You can download a full archive of your entries at any time.

| Format | URL | Description |
|--------|-----|-------------|
| JSON | [/api/journal/export/json](/api/journal/export/json) | All entries as a JSON array — includes slug, title, date, mood, tags, and content |
| Markdown | [/api/journal/export/markdown](/api/journal/export/markdown) | All entries as a single `.md` file — one `##` heading per entry with metadata and content |

Both downloads include only your own entries. The filename includes your username and today's date.

---

## Privacy and Security

- Entries are stored as **private pages** visible only to you and administrators.
- **Author-lock** (enabled by default) prevents other users from editing your entries even with a direct URL.
- Journal entries do **not** appear in site-wide search results for other users.
- Entries are excluded from public RSS feeds and page listings.

## API Reference

All endpoints require authentication.

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/journal` | Your entry timeline |
| `GET` | `/journal/new` | New entry editor form |
| `POST` | `/journal/new` | Save a new entry |
| `GET` | `/journal/:slug` | View a single entry |
| `GET` | `/journal/:slug/edit` | Edit form |
| `POST` | `/journal/:slug/edit` | Save changes |
| `POST` | `/journal/:slug/delete` | Delete an entry |
| `GET` | `/journal/tag/:tag` | Filter timeline by tag |
| `GET` | `/journal/mood/:mood` | Filter timeline by mood |
| `GET` | `/api/journal/new` | Create today's entry and redirect to editor |
| `GET` | `/api/journal/entries` | JSON list of your entries (supports `?limit=` and `?offset=`) |
| `GET` | `/api/journal/on-this-day` | JSON entries from the same calendar date in prior years |
| `GET` | `/api/journal/streak` | JSON `{ streak, total }` |
| `GET` | `/api/journal/templates` | JSON list of available writing templates |
| `GET` | `/api/journal/export/json` | Download all own entries as JSON |
| `GET` | `/api/journal/export/markdown` | Download all own entries as Markdown |
