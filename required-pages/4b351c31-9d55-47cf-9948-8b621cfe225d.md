# Overview

[{$applicationname}] caches rendered page content and other frequently accessed data in memory to improve response times. The **Clear Cache** action flushes all in-memory caches, forcing content to be re-rendered from source on the next request.

## When to Clear Cache

Clear the cache when:

- A template, plugin, or layout change is not reflecting on existing pages
- Required pages have been updated and the changes are not visible
- Stale search results appear after editing or deleting pages
- An administrator has changed system configuration that affects page rendering

Normal page edits automatically invalidate the cache for the affected page, so manual cache clearing is rarely needed.

## How to Clear Cache

1. Go to **Admin → Clear Cache**
2. Confirm the action when prompted
3. The server clears all caches and displays a confirmation message

The operation is fast and does not require a server restart. Users may notice slightly slower page loads for a short time while caches are rebuilt.

## What Is Cleared

- Rendered page HTML cache
- Plugin output cache
- Template fragment cache

The page index, search index, and user sessions are **not** affected by a cache clear.
