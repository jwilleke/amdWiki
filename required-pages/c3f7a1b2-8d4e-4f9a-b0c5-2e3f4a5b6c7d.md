# Using Private Keyword

The `private` [User Keywords] keyword gives a page a private storage location that is accessible only to admins and the page's creator. It is the strongest per-page privacy control available and bypasses the [Page Audience] setting entirely.

## How It Works

When you save a page with `private` in its `user-keywords`:

1. The page is moved to a **private storage location** separate from the normal page store
2. The [Page Audience] field is **bypassed entirely** — audience settings have no effect
3. Access is restricted to exactly two groups:
   - Users with the **`admin`** role
   - The page's **creator** — the user who first saved the page with the `private` keyword

> **Creator vs Author**: access uses the page's **creator** as recorded in the page index, not the `author` frontmatter field. If the `author` field differs from the actual creator, the `author` field is ignored for access control purposes.

## Applying the Private Keyword

In the editor, add `private` to the User Keywords field. In frontmatter:

```yaml
---
title: My Private Notes
system-category: documentation
user-keywords:
  - private
slug: my-private-notes
---
```

Only one keyword slot is consumed. You may combine `private` with up to four other keywords (the platform maximum is [{ConfigAccessor key="ngdpbase.maximum.user-keywords" valueonly=true}] keywords per page).

## What Private Does Not Do

- It does **not** hide the page from admins — admins always retain access
- It does **not** encrypt the page content — it is access-controlled, not encrypted
- It does **not** apply to attachments separately — attachment access follows the page
- Removing the `private` keyword makes the page visible under normal access policies again

## Private vs Audience

%%table-striped
|| Feature || Audience field || `private` keyword ||
| Where configured | Page metadata (Audience) | `user-keywords` frontmatter / editor |
| Who can view | Any roles/users listed | Admin + creator only |
| Storage location | Normal page store | Private location |
| Can be bypassed | By admin | No — admin always has access |
| Effect on other keywords | None | None |
/%

The `private` keyword takes precedence. If a page has both an Audience setting and the `private` keyword, the Audience setting is ignored.

See [Page Audience] for the full access-control model and role descriptions. See [User Keywords] for the complete list of available keywords and how they are configured.
