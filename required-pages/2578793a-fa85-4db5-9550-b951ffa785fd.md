# Delete Page

The **Delete Page** button permanently removes the current page from [{$applicationname}].

It appears in the toolbar only when you are on the edit form (`/edit/PageName`). It is visible to any user with edit permission for the page.

## How It Works

Clicking **Delete Page** shows a confirmation dialog: *"Are you sure you want to delete this page? This action cannot be undone."*

- Click **OK** to permanently delete the page and all its metadata. You are redirected to the home page after deletion.
- Click **Cancel** to return to editing without deleting.

## What Is Deleted

- The page content and frontmatter
- The page's entry in the page index

Attachments associated with the page are not automatically removed.

## Permissions

Only users with edit permission for the page can delete it. If [Author Lock] is set, only the page author and admins can delete the page.

Required pages (system-category `system` or `documentation`) should not be deleted — they will be re-created on the next Required Pages Sync.
