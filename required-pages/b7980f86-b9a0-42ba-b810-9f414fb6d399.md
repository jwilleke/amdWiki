# ConfigAccessorPlugin

Display configuration values, roles, permissions, and system settings on any page. See [Plugins] for a complete list of available plugins.

## Syntax

```
[{ConfigAccessor key='ngdpbase.server.port'}]
[{ConfigAccessor type='roles'}]
```

You must supply either `key` or `type`.

## Parameters

%%table-striped
|| Parameter || Description ||
| `key` | Dot-notation config key. Supports `*` wildcards. |
| `type` | Named display type — see types below. |
| `valueonly` | `true` — return plain text instead of a formatted card. |
| `before` | Prefix before each value (use with `valueonly`). |
| `after` | Suffix after each value (use with `valueonly`). Default: `''` for single values, `\n` for wildcards. |
| `manager` | Manager name — required when `type='manager'`. |
| `feature` | Feature name — required when `type='feature'`. |
| `enabled` | `true`/`false` — filter keywords or categories by enabled state. |
| `category` | Filter by category (use with `userKeywords`, `systemKeywords`). |
| `storageLocation` | `regular` or `required` — filter system categories. |
/%

## Notes

- `type='authMethods'` — sensitive fields (client-secret, tokens) are masked for non-admin users.
- Plugin name is case-insensitive: `[{configaccessor}]` and `[{ConfigAccessor}]` are equivalent.
- Pages using this plugin should be access-controlled if they expose sensitive config.
