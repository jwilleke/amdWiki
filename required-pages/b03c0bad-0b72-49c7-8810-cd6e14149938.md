# Page Audience

The **Audience** setting controls which roles can **view** a page. It is optional — leaving all roles unchecked means global access policies apply.

## Roles

| Role | Description |
|---|---|
| Admin | Full access — always able to view all pages |
| Editor | Can view and edit pages |
| Contributor | Can view pages and submit contributions |
| Reader | Can view pages only |
| Occupant | Minimal authenticated access |

## How It Works

- Check one or more roles to **restrict** view access to those roles only
- Leave **all unchecked** to apply the global access policies (the default)
- Checking a role does **not** grant edit access — only view access

## The `private` Keyword

If a page has the `private` [User Keywords] keyword, the `private` setting **overrides** the Audience field entirely — the page becomes visible only to admins and the page author regardless of what Audience is set to.
