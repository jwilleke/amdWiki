# [{$pagename}]

Fenced code blocks with a language specifier are rendered with syntax highlighting automatically.

## Description

Place a language name immediately after the opening triple backtick. The block is highlighted using [highlight.js|https://highlightjs.org/|target='_blank'] — no configuration required.

## Syntax

```
` ` `javascript
const greeting = 'hello';
` ` `
```

produces a highlighted JavaScript block. Replace `javascript` with any supported language name.

## Examples

JavaScript:

```javascript
function greet(name) {
  return `Hello, ${name}!`;
}
```

JSON:

```json
{
  "title": "Example",
  "enabled": true,
  "count": 42
}
```

YAML:

```yaml
ngdpbase:
  telemetry:
    enabled: true
    metrics:
      port: 9464
```

Bash:

```bash
./server.sh restart
npm run build
```

SQL:

```sql
SELECT username, email
FROM users
WHERE is_active = true
ORDER BY username;
```

A block with no language specifier renders as plain monospace — no highlighting applied:

```
This is plain text.
No language specified.
```

## Supported Languages

highlight.js supports over 190 languages. Commonly used ones on this platform:

%%table-striped
|| Specifier || Language ||
| `javascript` or `js` | JavaScript |
| `typescript` or `ts` | TypeScript |
| `json` | JSON |
| `yaml` | YAML |
| `bash` or `shell` | Shell / Bash |
| `sql` | SQL |
| `html` or `xml` | HTML / XML |
| `css` | CSS |
| `markdown` | Markdown |
| `python` | Python |

For the full list see the [highlight.js supported languages|https://highlightjs.org/|target='_blank'] page.

## Notes

- Language names are case-insensitive — ` ```JavaScript ` and ` ```javascript ` both work.
- Blocks without a language specifier degrade gracefully to plain monospace.
- Syntax highlighting is client-side only — it does not affect exported or printed output.
